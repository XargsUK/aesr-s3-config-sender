Instructions

What is AESR Config Sender?

AESR Config Sender is a web extension that enables seamless integration between your AWS Extend Switch Roles configuration and the corresponding extension. By capturing the SSO SAML response from AWS, the extension exchanges it for tokens used to retrieve your configuration from an S3 source.

Utilizing different S3 profiles, AESR Config Sender downloads the necessary configuration and, with the AESR extension ID, pushes it directly to the AWS Extend Switch Roles extension.

Initial Setup

Begin by navigating to the Sync section and enter the extension ID of the AWS Extend Switch Roles. This links AESR Config Sender with AWS Extend Switch Roles, allowing configurations to be pushed directly.

Use the 'Download Config' button to fetch the configuration file from S3. This populates the 'Configuration from S3' field with the configuration text.

Click 'Send to AESR' to transfer the configuration from the text field to the AWS Extend Switch Roles extension.

In the Settings section, define your profiles and S3 details to manage where and how configurations are sourced and sent.

Settings Explained
Profile Name:
The user-defined name for the profile being configured.
Region:
The AWS region where the S3 bucket is located.
Bucket Name:
The name of the S3 bucket that contains the AWS Extend Switch Roles configuration.
File Key:
The specific file path (key) within the S3 bucket where the configuration file resides.

Enabling 'Debug Mode' will log extension activities to the console, aiding in troubleshooting.

Profile Management

Use 'Import Profile' to bring a JSON-formatted profile into the extension's profile list, and 'Export Profile' to save the current profile as a JSON file for backup or transfer purposes.

Linking with AWS Extend Switch Roles

To complete the setup, open the AWS Extend Switch Roles extension and navigate to the section labeled 'Config sender extension ID'. Enter the ID of the AESR Config Sender extension there to establish a connection.
